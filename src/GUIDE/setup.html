Hướng Dẫn Cài Đặt & Khởi Chạy

Quy trình triển khai máy chủ Node.js REST API và thiết lập mã nguồn ứng dụng di động Kotlin Android cục bộ.

Yêu Cầu Cài Đặt Trước (Prerequisites)

Đảm bảo máy tính của bạn đã được cài đặt sẵn các môi trường phát triển sau:

  • Node.js: Phiên bản 18.x trở lên (Khuyến nghị LTS mới nhất) và NPM package manager.
  • MongoDB: Bản cài đặt cục bộ MongoDB Community Edition (chạy cổng mặc định 27017) hoặc tài khoản MongoDB Atlas Cloud.
  • Android Studio: Phiên bản Ladybug/Iguana trở lên để biên dịch mã nguồn Kotlin.
  • JDK 17: Môi trường chạy Java cần thiết cho Gradle biên dịch Android.

Hướng Dẫn Cấu Hình Máy Chủ Back-End

Thực hiện lần lượt các bước sau trong cửa sổ terminal của dự án backend:

Bước 1: Cài Đặt Thư Viện

Truy cập thư mục backend F:\backend-vcar và tiến hành cài đặt toàn bộ gói phụ thuộc (dependencies):

cd F:\backend-vcar
npm install

Bước 2: Cấu Hình Môi Trường (.env)

Tạo file .env ở thư mục gốc của dự án backend dựa trên mẫu .env.example và cấu hình như sau:

PORT=5000
NODE_ENV=development
MONGODB_URI=mongodb://localhost:27017/car-rental-mvp
JWT_SECRET=vcar_super_secure_jwt_token_2026_key
JWT_EXPIRES_IN=30d

Bước 3: Khởi Tạo Cơ Sở Dữ Liệu Gốc (Database Seeding)

Để tự động tạo cấu trúc và nạp dữ liệu mẫu ban đầu (như xe hạng sang, tài khoản mẫu và tải trước tài nguyên ảnh tĩnh về thư mục uploads), chạy lệnh:

npm run seed

Kết Quả Nạp Dữ Liệu Seeder

Lệnh này sẽ wipe sạch cơ sở dữ liệu cũ, khởi tạo 6 tài khoản người dùng, nạp danh sách các xe có sẵn, và tải ảnh thực tế từ internet lưu vào thư mục cục bộ public/uploads/ để phục vụ hiển thị trên app di động.

Thông Tin Tài Khoản Mẫu Mặc Định

Các tài khoản được khởi tạo tự động trong hệ thống mà bạn có thể dùng để đăng nhập kiểm thử:

Tên Người Dùng Email / Tài Khoản Mật Khẩu Vai Trò (Role)

Nguyễn Văn A john@gmail.com password123 Khách hàng (User)

Bước 4: Khởi Chạy Máy Chủ

Để khởi chạy máy chủ dưới chế độ phát triển (tự động tải lại mã nguồn khi thay đổi nhờ Nodemon):

npm run dev

Khi thành công, terminal sẽ in ra log:

[Database] Connected successfully to MongoDB
[Server] Running on port 5000 in development mode
[Cleanup Service] Auto-cleanup expired bookings initialized...

Hướng Dẫn Cấu Hình Front-End Android Client

Để ứng dụng di động có thể gọi API từ máy chủ cục bộ của bạn, thực hiện cấu hình như sau:

Bước 1: Mở Dự Án Bằng Android Studio

Khởi chạy Android Studio, chọn Open và trỏ tới thư mục e:\CodeMobile\Vcar. Chờ Gradle đồng bộ hóa các thư viện (Gradle Sync) hoàn tất.

Bước 2: Thay Đổi Địa Chỉ IP Máy Chủ (IP Mapping)

Để Retrofit kết nối tới máy chủ backend nội bộ của bạn, hãy mở file cấu hình liên kết mạng tại đường dẫn:

RetrofitClient.kt

Sửa hằng số BASE_URL cho phù hợp với môi trường thử nghiệm của bạn:

1. Nếu Chạy Trên Máy Ảo (Android Emulator)

Sử dụng IP loopback mặc định của máy ảo Android để giao tiếp với máy vật lý:

private const val BASE_URL = "http://10.0.2.2:5000/"

2. Nếu Chạy Trên Thiết Bị Thật (Physical Device)

Tìm địa chỉ IP nội bộ (LAN IP) của máy tính của bạn (bằng lệnh ipconfig trên CMD của Windows, ví dụ 192.168.1.15), đảm bảo điện thoại và máy tính kết nối chung mạng Wi-Fi:

private const val BASE_URL = "http://192.168.1.15:5000/"

Chú Ý Quan Trọng Về Giao Thức HTTP

Bắt buộc phải giữ giao thức http:// ở đầu URL nội bộ thay vì https://. Ứng dụng đã được kích hoạt cấu hình android:usesCleartextTraffic="true" trong file AndroidManifest.xml để cho phép truyền tải dữ liệu dạng HTTP không mã hóa trong môi trường local.

Bước 3: Biên Dịch Và Khởi Chạy App

  • Nhấn nút Sync Project with Gradle Files để tải các thư viện.
  • Kết nối thiết bị hoặc khởi chạy Emulator.
  • Nhấn nút **Run 'app'** (phím tắt `Shift + F10`) để cài đặt ứng dụng lên thiết bị.

Khắc Phục Lỗi Thường Gặp (Troubleshooting)

1. Không Kết Nối Được API Trên Điện Thoại Thật

Triệu chứng: Ứng dụng Android load vô tận hoặc báo lỗi kết nối mạng (SocketTimeoutException / ConnectException).

Giải pháp:

  1. Kiểm tra xem máy tính và điện thoại đã kết nối chung một điểm phát Wi-Fi chưa.
  2. Tắt tạm thời tường lửa Windows (Windows Firewall) hoặc thêm quy tắc loại trừ (Inbound Rule) cho phép truy cập cổng TCP 5000.
  3. Xác nhận lại địa chỉ IP trong RetrofitClient.kt bằng cách chạy ipconfig để tránh trường hợp Router Wi-Fi cấp phát IP mới cho máy tính.

2. MongoDB Không Thể Kết Nối

Triệu chứng: Khi start backend báo lỗi MongooseServerSelectionError.

Giải pháp:

  • Mở CMD và gõ lệnh services.msc để kiểm tra dịch vụ MongoDB Database Server đã ở trạng thái Running chưa. Nếu chưa, click chuột phải chọn **Start**.
  • Đảm bảo cổng 27017 không bị chiếm dụng bởi phần mềm khác.

3. Lỗi Đăng Nhập Không Hợp Lệ

Triệu chứng: Nhập đúng tài khoản admin hoặc email john nhưng server báo sai thông tin xác thực.

Giải pháp: Chạy lại lệnh seeder npm run seed để làm sạch và khôi phục các tài khoản và mật khẩu mặc định của hệ thống về đúng định dạng lưu trữ đã băm.
