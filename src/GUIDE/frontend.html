Front-End Mobile Developer Guide

Hướng dẫn kiến trúc mã nguồn Kotlin Android, cấu hình Retrofit, định dạng layout và luồng đăng ký thuê xe.

Kiến Trúc Dự Án Android

Dự án di động Android được viết bằng ngôn ngữ Kotlin, sử dụng mô hình MVC tối giản kết hợp với các thư viện kết nối mạng và tải ảnh hiện đại.

Cấu Trúc Gói Lập Trình (Packages)

com.example.vcar.
├── activities/       # Quản lý tất cả màn hình giao diện (Splash, Login, Home, v.v.)
├── adapter/          # Bộ liên kết dữ liệu cho RecyclerView: CarAdapter, BookingAdapter
├── model/            # Khai báo các đối tượng dữ liệu Kotlin (Data Classes)
├── network/          # Cấu hình Retrofit API Client và đặc tả các yêu cầu (Requests)
└── utils/            # Các tiện ích lưu trữ cục bộ SharedPrefManager để lưu JWT Token

Các Thư Viện Cốt Lõi (build.gradle)

  • Retrofit 2 & Converter Gson: Dùng để gọi API không đồng bộ và tự động parse JSON sang Model Class.
  • Glide 4: Tải ảnh xe mượt mà từ máy chủ web và tự động lưu cache.
  • Material Components: Dùng để vẽ các nút bấm bo góc, hộp thoại thông báo cao cấp.
  • ConstraintLayout: Giúp thiết kế giao diện phẳng, tối ưu hóa hiệu năng render UI trên điện thoại.

Đặc Tả Các Màn Hình Chính

1. SplashActivity (Màn hình mở đầu)

Hiển thị ảnh nền dọc 9:16 chất lượng cao chống biến dạng bằng cách sử dụng thẻ ImageView có thuộc tính scaleType="centerCrop".
Logic điều hướng: Đọc Token đã lưu trong SharedPrefManager:

  • Nếu có Token: Giữ màn hình 1.5 giây và tự động chuyển thẳng vào màn hình chính HomeActivity (Auto-login).
  • Nếu không có Token: Hiển thị bộ nút hành động và liên kết Đăng Nhập kèm hiệu ứng chuyển động mờ (fade-in) 800ms.

2. HomeActivity (Trang chủ danh sách xe)

Danh sách xe được trình bày dạng thẻ kính mờ (Glassmorphism) với đường viền mảnh. Mỗi thẻ xe (item_car.xml) bao gồm:

  • Ảnh xe nằm nổi (inset margin 12dp) có bo góc 16dp.
  • Huy hiệu tình trạng xe (Còn xe / Hết xe) sử dụng hình nền mờ có màu sắc riêng biệt (Available / Rented) được gán linh động từ adapter.
  • Lưới thông số (Spec Grid): Đời xe, Số tự động, Nhiên liệu xăng.
  • Giá thuê nổi bật màu xanh Cyan ở chân thẻ.

3. BookingActivity (Màn hình đặt xe)

Màn hình chọn ngày đặt xe và hiển thị hộp thoại xác nhận điều khoản:

  • Xử lý Điều Khoản Dịch Vụ: Khi nhấn nút "Đặt xe ngay", ứng dụng hiển thị hộp thoại MaterialAlertDialog yêu cầu chấp thuận các điều khoản thuê xe và hướng dẫn bắt buộc phải thực hiện thêm 1 bước gọi trực tiếp đến số Hotline 0363970865 để nhân viên xác nhận.
  • Đọc mã lỗi Server: Khi gửi yêu cầu đặt xe, nếu server trả về mã lỗi (ví dụ do trùng đơn hàng), ứng dụng sẽ tự động trích xuất chuỗi JSON trong errorBody() để hiển thị Toast mô tả lỗi chính xác thay vì báo lỗi chung chung.

4. HistoryActivity (Lịch sử đặt xe)

RecyclerView hiển thị các đơn hàng của khách hàng cùng bộ đếm ngược thời gian (Countdown Timer) trong BookingAdapter.kt: 

// Tính thời gian đếm ngược còn lại của đơn thuê xe (Tối đa 1 ngày)
val baseDate = if (status == "approved") parse(rentalDate) else parse(createdAt)
val expirationTimeMs = baseDate.time + (24 * 60 * 60 * 1000)
val remainingMs = expirationTimeMs - System.currentTimeMillis()

if (remainingMs > 0) {
    val hours = remainingMs / (60 * 60 * 1000)
    val minutes = (remainingMs % (60 * 60 * 1000)) / (60 * 1000)
    txtRemainingTime.text = "Còn lại: " + hours + "h " + minutes + "m"
} else {
    txtRemainingTime.text = "Đã hết hạn"
}
