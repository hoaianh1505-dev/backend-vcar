Back-End Developer Guide

Chi tiết thiết kế, mô hình cơ sở dữ liệu và đặc tả REST API hệ thống máy chủ Vcar.

Công Nghệ Sử Dụng & Cấu Trúc Thư Mục

Mã nguồn máy chủ được xây dựng trên môi trường Node.js sử dụng framework Express, cơ sở dữ liệu phi quan hệ MongoDB (kết nối thông qua Mongoose ORM).

Cấu Trúc Thư Mục Core

F:\backend-vcar\src\
├── config\            # Cấu hình kết nối CSDL và dữ liệu mẫu seeder
├── controllers\       # Điều phối logic nghiệp vụ REST API
├── docs\              # Tích hợp tài liệu tự động Swagger
├── middlewares\       # Bộ lọc trung gian: Xác thực JWT, Xử lý lỗi, Uploads
├── models\            # Định nghĩa các lược đồ thực thể MongoDB (Mongoose Schema)
├── routes\            # Định tuyến các endpoint HTTP
├── services\          # Nghiệp vụ xử lý logic nghiệp vụ sâu (Ví dụ: đặt xe, dọn dẹp)
├── views\             # Giao diện quản trị Admin Panel hiển thị bằng EJS
├── app.js             # Cấu hình Middleware toàn cục của Express
└── server.js          # Điểm khởi động máy chủ & Lập lịch tác vụ nền

Thiết Kế Mô Hình Dữ Liệu (Schemas)

Cơ sở dữ liệu MongoDB gồm 3 collection chính được định nghĩa trong thư mục src/models/:

1. User (Tài Khoản)

Lưu trữ tài khoản người dùng và thông tin phân quyền:

  • fullName (String, required)
  • email (String, unique, required) - Dùng để đăng nhập
  • phone (String, required)
  • password (String, required) - Lưu trữ mật khẩu đã băm (Bcrypt)
  • role (String, default: 'user') - Quyền hạn: 'user' hoặc 'admin'

2. Car (Phương Tiện)

Lưu trữ thông tin chi tiết xe và tình trạng sẵn có:

  • name (String, required) - Tên mẫu xe (Ví dụ: Civic, VF9)
  • brand (String, required) - Hãng xe (Ví dụ: Honda, Vinfast)
  • year (Number, required) - Năm sản xuất
  • pricePerDay (Number, required) - Giá thuê theo ngày
  • description (String, required) - Mô tả chi tiết xe
  • location (String, required) - Địa bàn showroom cho thuê
  • images (Array of String) - Đường dẫn ảnh tĩnh của xe
  • available (Boolean, default: true) - Tình trạng xe còn trống hay đang được thuê

3. Booking (Đơn Thuê Xe)

Lưu trữ liên kết đặt thuê xe và quản lý bộ thời gian:

  • userId (ObjectId, ref: 'User') - ID tài khoản thuê
  • carId (ObjectId, ref: 'Car') - ID xe được thuê
  • rentalDate (Date, required) - Ngày bắt đầu thuê xe
  • status (String, enum: ['pending', 'approved', 'cancelled']) - Tình trạng đơn
  • timestamps (Boolean: true) - Tự động ghi nhận thời gian tạo đơn createdAt

Đặc Tả API Điển Hình Phía Ứng Dụng Di Động

Các API được thiết kế theo chuẩn RESTful, tất cả các yêu cầu gửi lên cần kèm theo Header: Authorization: Bearer <Token>

POST /api/auth/register

Đăng ký tài khoản người dùng mới. Mã hóa mật khẩu.

POST /api/auth/login

Xác thực tài khoản và trả về chuỗi Token JWT (hạn 30 ngày) cùng thông tin cơ bản.

POST /api/bookings

Gửi yêu cầu đặt xe.
Logic Ràng Buộc: Server sẽ thực hiện truy vấn các đơn đặt có trạng thái pending hoặc approved của người dùng. Nếu phát hiện đã có đơn đang chạy, hệ thống sẽ từ chối và trả về lỗi 400 Bad Request để thực thi quy tắc giới hạn thuê xe 1 ngày/lượt.

GET /api/bookings/my

Tải về lịch sử và các đơn đặt xe của người dùng hiện tại, tự động nạp (populate) chi tiết xe kèm theo.

Tác Vụ Dọn Dẹp Cơ Sở Dữ Liệu Tự Động (Background Job)

Trong file src/services/booking.service.js, tác vụ autoCancelExpiredBookings() được định cấu hình khởi chạy ngay khi bật máy chủ và lặp lại định kỳ mỗi 1 giờ một lần (sử dụng setInterval): 

// Định nghĩa tác vụ nền dọn dẹp đơn cũ quá 1 ngày
export const autoCancelExpiredBookings = async () => {
  try {
    const oneDayAgo = new Date(Date.now() - 24 * 60 * 60 * 1000);
    
    // Tìm các đơn đặt xe có thời gian tạo hoặc ngày thuê quá 24 giờ trước
    const expiredBookings = await Booking.find({
      $or: [
        { createdAt: { $lt: oneDayAgo } },
        { rentalDate: { $lt: oneDayAgo } }
      ]
    });

    if (expiredBookings.length > 0) {
      const bookingIds = expiredBookings.map(b => b._id);
      const carIds = expiredBookings.map(b => b.carId);

      // Xóa vĩnh viễn các đơn hàng hết hạn khỏi Database
      await Booking.deleteMany({ _id: { $in: bookingIds } });

      // Giải phóng xe: cập nhật lại available = true để người khác thuê
      await Car.updateMany(
        { _id: { $in: carIds } },
        { $set: { available: true } }
      );
    }
  } catch (error) {
    console.error('[Cleanup Error]', error.message);
  }
};
